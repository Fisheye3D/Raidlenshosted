## ✨ Features

| Tab | What it shows |
|-----|---------------|
| 📈 **Summary** | Raid scorecard — outcome, deaths, interrupts, pots, avg DPS, overheal, missed CDs, per-player grades |
| 🛡️ **Defensives** | Every defensive cooldown used per player with cast count |
| ⚔️ **Offensive CDs** | Major DPS cooldowns used ✓ and missed ✗ per spec |
| 🧪 **Consumables** | Potion (with count) and Healthstone usage per player |
| 💀 **Deaths** | Every death in order with timestamp and killing blow — survivors marked ✓ |
| 🎯 **Mechanic Deaths** | Deaths grouped by the ability that killed each player |
| ⚡ **Interrupts** | Which spells each player interrupted and how many times |
| ✨ **Dispels** | Which debuffs each player dispelled and how many times |
| 🛑 **Crowd Control** | Stuns, roots, incapacitates, and slows used by each player |
| 💥 **Damage Taken** | Total damage + DTPS + % of raid per player with role color bar chart |
| ⚔️ **DPS** | Total damage dealt and DPS for every player with bar chart |
| 📊 **DPS Breakdown** | Full DPS table — damage, deaths, CDs used/missed, potion, and Sim button |
| 💚 **HPS** | Total healing done and HPS for every healer with bar chart |
| 💊 **Healing Breakdown** | Effective healing, overheal %, absorbs, and HPS per healer |
| 🔄 **CD Timeline** | Visual timeline of every major cooldown pressed across the fight |
| ⚡ **Bloodlust** | When it was cast, who cast it, and how many players were alive to benefit |
| ⚠️ **Wipe Summary** | One-click Discord-ready pull summary with deaths, bloodlust, and CDs |
| 📜 **Casts** | Every spell cast per player — click to expand full list sorted by most used |
| 🛡️ **Tank Threat** | Which tank held which boss and for how long, with % and duration bars |
| ⏱️ **Downtime** | Seconds each DPS player spent doing zero damage with green/red activity bar |
| 🔁 **Pull Compare** | Side-by-side DPS, deaths, and missed CD comparison across attempts |
| 👤 **Player Modal** | Click any player for full breakdown plus Raider.io, Raidbots sim, and Armory links |

**Role colors throughout:** 🟡 Tank · 🟢 Healer · 🔵 DPS

**Other features:**
- ⭐ **My Logs** — reports auto-fetch and save on every load (Personal Profile and Guild modes)
- 🔄 **Auto-refresh** — token renews automatically, no manual re-entry needed
- 🔍 **Guild Search tab** — search any guild's public logs at any time
- ↕️ **Sortable columns** — click any column header to sort
- 🎨 **Class colors** — every player name colored by their WoW class
- 🎮 **Sim on Raidbots** — one-click sim button on every player, pre-filled with their character
- ⚔️ **WoW Armory links** — direct armory link on every player card and modal
- 🏅 **Raider.io integration** — M+ score, item level, and raid progression shown per player
- 🔒 **Nothing server-side** — credentials and data stay on your computer only

## 🚀 First-Time Setup

When you first open RaidLens you will see a one-time setup screen. You only do this once.

### Step 1 — Get your free API credentials

1. Go to **https://www.warcraftlogs.com/api/clients/**
2. Log in to your WarcraftLogs account
3. Click **"Create Client"**
4. Fill in:
   - **Name:** anything you like, e.g. `RaidLens`
   - **Redirect URL:** `https://localhost`
   - Leave **Public Client** unchecked
5. Click **Create**
6. Copy your **Client ID** and **Client Secret**

### Step 2 — Paste them into RaidLens

Open RaidLens, paste both values into the setup screen. No terminal, no commands.

### Step 3 — Choose how to load reports

| Option | How it works |
|--------|-------------|
| 👤 Personal Profile | Paste your WarcraftLogs profile URL — auto-fetches your reports on every load |
| 🏰 Guild Logs | Enter guild name, server, and region — auto-fetches guild reports on every load |
| 🔗 Paste URL | Manually paste a report URL when you need it |

### Finding Your Profile URL

1. Go to **https://www.warcraftlogs.com**
2. Click your name top-right → **My Reports**
3. Copy the URL — it looks like: `https://www.warcraftlogs.com/user/reports-list/123456`
4. Paste that into the Personal Profile field

After setup, settings are saved locally. Reports load automatically every time.

## ⚠️ Notes

- **Aug Runes** are applied as a pre-fight buff — not detectable via the cast log API
- **Healthstone** detection uses known spell IDs — new patch IDs may need manual updating
- **Avoidable damage** is not tracked — WarcraftLogs does not tag boss abilities this way via the API
- **Personal logs** must be set to **public** or **unlisted** on WarcraftLogs for auto-fetch to work
- **Tank Threat** is estimated from incoming damage events — it approximates tanking time, not actual threat values
- **Pull Compare** resets when you close the browser tab — it is session-only
- All data fetched directly from WarcraftLogs — nothing stored server-side
